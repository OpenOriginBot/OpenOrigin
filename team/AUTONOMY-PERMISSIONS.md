# AUTONOMY-PERMISSIONS.md - 自主权限分级

> 划定各主管无需请示即可自主决策的操作范围，提高响应效率。

---

## 一、权限分级定义

### 🟢 绿色权限（立即执行）

**定义：** 内部事务、状态更新、调研工作等低风险操作。

**特点：**
- 无需请示，立即执行
- 执行后直接更新看板/记录
- 无需额外审批

**范围：**

| 类别 | 具体操作 |
|------|----------|
| **状态更新** | 任务状态变更（不涉及结果判定） |
| | 备注添加 |
| | 进度更新 |
| **内部查询** | 库存查询 |
| | 订单状态查询 |
| | 数据统计查询 |
| **文档整理** | 内部文档更新 |
| | 会议纪要整理 |
| | 资料归档 |
| **信息通知** | 内部进度通报 |
| | 状态变更通知 |
| **调研分析** | 竞品信息收集（不公开） |
| | 行业动态分析（不公开） |
| | 数据分析（不出外部） |

---

### 🟡 黄色权限（执行后报备）

**定义：** 任务办结、初稿撰写等对业务有直接影响但不不可逆的操作。

**特点：**
- 可立即执行
- 执行后必须更新看板/记录
- 24小时内向相关方报备
- 如有异议，可回溯调整

**范围：**

| 类别 | 具体操作 |
|------|----------|
| **内容发布** | 详情页文案初稿 |
| | 产品主图（非最终版） |
| | 推广文案初稿 |
| **方案撰写** | 报价方案初稿（未给客户前） |
| | 项目提案初稿（未给客户前） |
| | 内部流程文档 |
| **任务闭环** | 任务标记为「已完成」 |
| | 委派任务确认完成 |
| | 验收通过确认 |
| **客户沟通** | 标准回复模板发送 |
| | 催发货标准话术 |
| | 常规咨询回复 |
| **数据调整** | 内部数据修正 |
| | 库存数据校准 |
| **资源调配** | 内部工作分配调整 |
| | 优先级调整（内部） |

---

### 🔴 红色权限（事前请示）

**定义：** 对外沟通、费用支出、不可逆变更等高风险操作。

**特点：**
- 必须事前请示 CEO 或人类负责人
- 获得明确批准后方可执行
- 需保留审批记录
- 如紧急情况，先执行后1小时内报备

**范围：**

| 类别 | 具体操作 |
|------|----------|
| **对外沟通** | 给客户的正式邮件/回复 |
| | 对外报价（任何金额） |
| | 客户投诉升级处理方案 |
| | 公开内容发布 |
| **费用支出** | 退款审批（任何金额） |
| | 优惠发放（任何金额） |
| | 采购决策（任何金额） |
| | 合同签署 |
| **不可逆变更** | 删除订单 |
| | 修改客户数据 |
| | 修改历史财务数据 |
| | 供应商更换 |
| **权限变更** | 子智能体新增/删除 |
| | 定时任务修改 |
| | 委派规则修改 |
| **敏感操作** | 银行卡/支付信息查看 |
| | 账号密码修改 |
| | 系统配置变更 |
| | API密钥变更 |

## 三、权限申请流程

### 🔴 红色权限申请

```
1. 识别为红色权限操作
   ↓
2. 准备申请材料：
   - 操作内容
   - 原因/背景
   - 预期结果
   - 风险评估
   - 备选方案
   ↓
3. 提交 CEO / 人类负责人
   ↓
4. 等待审批
   - 紧急：1小时内回复
   - 一般：4小时内回复
   ↓
5. 获得批准？
   - 是 → 执行 + 记录
   - 否 → 不执行 + 记录原因
```

### 紧急情况处理

如遇紧急情况（如客户强烈投诉），可：
1. 先执行必要措施
2. 1小时内补报 CEO
3. 提供情况说明

## 四、权限监管

### 定期抽查

- CEO 每日志疑异权限使用情况
- 每周审核权限变更记录
- 每月评估权限设置合理性

### 违规处理

| 违规类型 | 处理方式 |
|----------|----------|
| 无意违规 | 记录 + 提醒 |
| 重复违规 | 记录 + 警告 |
| 造成损失 | 记录 + 上报 + 整改 |
