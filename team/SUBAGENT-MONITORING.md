# SUBAGENT-MONITORING.md - 子智能体监管规则

> 规范子智能体运行监管，确保响应质量、故障及时发现、工作不被替代。

---

## 一、响应超时判定

### 判定标准

| 条件 | 说明 |
|------|------|
| **最短等待时间** | 10 分钟（600秒） |
| **更新时间戳** | 任务下发后系统自动记录 |
| **空白消息** | 不计入有效响应，忽略计时 |
| **进度更新** | 有实质内容的更新才算有效响应 |

### 超时阈值

| 任务类型 | 超时阈值 |
|----------|----------|
| 简单查询 | 5 分钟 |
| 标准任务（≤200字） | 10 分钟 |
| 复杂任务（文档生成） | 20 分钟 |
| 批量任务 | 30 分钟 |

### 超时处理流程

```
任务派发 → 启动计时
    ↓
是否收到有效响应？
    ↓ 是
处理响应 → 完成任务
    ↓ 否
等待时间 ≥ 10分钟？
    ↓ 否
继续等待
    ↓ 是
是否达到超时阈值？
    ↓ 否
继续等待（最多再等10分钟）
    ↓ 是
判定为「响应超时」
    ↓
终止当前执行
    ↓
上报异常（不代为处理）
    ↓
记录日志 → 任务状态更新为「待处理」
```

## 二、故障判定与处理

### 故障类型

| 故障类型 | 判定条件 | 处理方式 |
|----------|----------|----------|
| 响应超时 | 超过阈值无有效响应 | 上报 + 终止 |
| 执行异常 | 进程崩溃/报错 | 上报 + 记录 |
| 死循环 | CPU占用>80%持续>5分钟 | 强制终止 + 上报 |
| 内存泄漏 | 内存占用>1GB | 强制终止 + 上报 |

### 故障上报模板

```
【子智能体故障报告】
时间：[YYYY-MM-DD HH:mm:ss]
子智能体：[子智能体名称/ID]
故障类型：[响应超时/执行异常/死循环/内存泄漏]
任务编号：[正在处理的任务]
错误信息：[如有]
已耗时：[从派发到故障的时间]
处理结果：[已终止/已重启/待人工处理]
建议：[后续处理建议]
```

## 三、禁止行为（红线）

### ❌ 绝对禁止

1. **代为处理工作**
   - 子智能体无响应时，不得代替其执行任务
   - 仅上报异常，不做实质性处理

2. **强制干预进程**
   - 除非超时或故障，不得强制终止子智能体
   - 不得修改子智能体的工作空间文件

3. **隐瞒故障**
   - 所有故障必须上报
   - 不得自行消化或忽略异常

4. **伪造响应**
   - 不得以主控身份伪造子智能体响应
   - 仅如实传递子智能体的输出

## 四、运行质量标准

### 有效响应判定

以下内容算作有效响应：
- ✅ 包含实质性工作输出的消息
- ✅ 明确的进度更新（百分比/状态）
- ✅ 请求澄清的合理问题
- ✅ 遇到阻滞的明确说明

以下内容不算作有效响应：
- ❌ 仅「收到」
- ❌ 仅「处理中」
- ❌ 仅表情/符号
- ❌ 空白消息

### 性能基线

| 指标 | 期望值 | 告警阈值 |
|------|--------|----------|
| 响应率 | ≥95% | <90% |
| 平均响应时间 | <10分钟 | >15分钟 |
| 任务完成率 | ≥90% | <85% |
| 误报率 | <5% | >10% |

## 五、监控记录

### 每次任务必须记录

```
任务编号：
派发时间：
派发主管：
执行子智能体：
任务类型：
期望完成时间：
首次响应时间：
任务完成时间：
处理结果：[成功/失败/超时/终止]
质量评分：[1-5]
备注：
```

### 每日汇总

每日 22:00 自动生成子智能体运行汇总：
- 任务总数
- 成功/失败/超时数量
- 平均响应时间
- 故障列表
- 改进建议

## 六、异常升级路径

```
子智能体异常
    ↓
主控智能体接收告警
    ↓
评估是否需要人工介入
    ↓
    ├── 不需要 → 记录日志，任务重新派发
    │         或标记为「待处理」
    │
    └── 需要 → 上报人类（老板）
              ↓
          提供：故障报告 + 建议方案
          等待：人工决策
```

---

## 七、只反馈已完成，不反馈已派发

### 正确示范

```
✅ 任务001已完成：发货时效优化方案已提交
✅ 任务002进行中：60%，预计30分钟完成
✅ 任务003失败：供应商数据无法获取，建议切换备选供应商
```

### 错误示范

```
❌ 任务001已派发
❌ 任务002已交给xxx处理
❌ 任务003正在处理
```
