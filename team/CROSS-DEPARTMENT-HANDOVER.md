# CROSS-DEPARTMENT-HANDOVER.md - 跨部门交接约定

> 规范跨部门工作流转的权责移交、资料交接、验收标准和反馈机制。

---

## 一、权责移交规则

### 移交原则

1. **责任不真空** — 任何时刻必须有且仅有一个责任方
2. **知情不替代** — 接收方知情但不替代原责任方
3. **闭环可追溯** — 全流程留痕，可回溯检查

### 权责矩阵

| 阶段 | 责任方 | 协作方 | 监督方 |
|------|--------|--------|--------|
| 任务发起 | 发起主管 | 接收主管 | CEO |
| 执行中 | 接收主管 | 发起主管 | — |
| 办结确认 | 发起主管 | 接收主管 | — |
| 事后复盘 | 双方+CEO | — | — |

### 交接触发条件

以下情况需要触发跨部门交接：

1. 任务类型跨越部门职责边界
2. 单一任务需要多部门协同
3. 前置任务完成后触发后续任务
4. 任务执行中遇阻需要其他部门资源

## 二、必备背景资料包

### 标准交接包

当一个任务从 A 主管交接给 B 主管时，必须提供：

#### 基础信息
```
【任务交接单】
任务编号：[自动生成]
交接时间：[YYYY-MM-DD HH:mm]
发起主管：[A主管名称]
接收主管：[B主管名称]
```

#### 背景资料
```
任务描述：
[清晰描述需要完成什么]

已完成工作：
[如有，列出已完成的部分及产出物]

待完成工作：
[详细列出还需要做什么]

关键信息：
- 客户/供应商：[名称]
- 截止时间：[YYYY-MM-DD HH:mm]
- 特殊要求：[如有]
```

#### 验收标准
```
办结判定条件：
1. [具体可衡量的标准]
2. [具体可衡量的标准]
3. [具体可衡量的标准]

不达标处理：
[如不符合，如何处理]
```

### 各场景资料包

**场景1：运营 → 营销（产品上新）**
- 产品规格文档
- 竞品分析参考
- 目标定价区间
- 上市时间要求

**场景2：营销 → 运营（客户投诉升级）**
- 客户联系方式
- 投诉内容摘要
- 已处理记录
- 客户诉求层级

**场景3：营收 → 运营（采购需求）**
- 采购清单
- 预算范围
- 供应商候选
- 期望到货时间

**场景4：任意 → 营收（成本核算）**
- 任务背景
- 数据来源
- 分析目的
- 截止时间

## 三、交接执行流程

```
┌─────────────────────────────────────────────────────┐
│                    发起方                              │
│  1. 确认任务需要跨部门                               │
│  2. 准备交接包                                       │
│  3. 发起交接申请                                     │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│                    接收方                              │
│  4. 确认接收 / 拒绝 / 协商                           │
│     - 拒绝需说明理由                                 │
│     - 协商需提出条件                                 │
└─────────────────────┬───────────────────────────────┘
                      │
                      ▼
┌─────────────────────────────────────────────────────┐
│                    双方确认                            │
│  5. 权责正式移交                                     │
│  6. 更新任务状态                                     │
│  7. 启动执行                                         │
└─────────────────────────────────────────────────────┘
```

## 四、交付验收标准

### 验收判定

| 场景 | 验收方 | 验收标准 |
|------|--------|----------|
| 营销素材交付 | 运营 | 符合规格+按时交付 |
| 数据分析交付 | 营收 | 数据准确+格式规范 |
| 客户方案交付 | 营销 | 客户认可+成本合理 |
| 采购完成交付 | 运营 | 到货验收+质量合格 |

### 验收流程

```
接收方完成工作
    ↓
通知发起方验收
    ↓
发起方核查
    ├── 合格 → 确认闭环
    │         ↓
    │      更新任务状态为「已完成」
    │
    └── 不合格 → 发起整改要求
                 ↓
              接收方整改
                 ↓
              重新验收（最多3次）
```

### 验收不通过处理

| 次数 | 处理方式 |
|------|----------|
| 第1次 | 整改+记录 |
| 第2次 | 整改+原因分析 |
| 第3次 | 上报CEO仲裁 |

## 五、反馈沟通机制

### 定期反馈

| 任务周期 | 反馈频率 | 反馈内容 |
|----------|----------|----------|
| ≤1天 | 完成时 | 结果汇报 |
| 1-3天 | 每24小时 | 进度简报 |
| 3-7天 | 每12小时 | 进度+阻滞 |
| >7天 | 每日站会 | 进度+风险 |

### 反馈格式

```
【跨部门任务进度报告】
任务：[编号+名称]
当前状态：[执行中/遇阻/已完成]
进度：[百分比/完成度]
阻滞：[如有]
预计完成：[时间]
需协助：[如有]
```

### 紧急反馈

以下情况立即反馈：
- 发现可能延期（超过原计划50%时间）
- 发现重大风险
- 需要追加资源
- 已完成/已失败

## 六、跨部门任务看板

所有跨部门任务在项目管理看板上：
- 标注「跨部门」
- 显示当前责任方
- 显示交接历史
- 显示验收状态

状态标签：
- 🔄 进行中（交接完成）
- ⏸️ 等待交接
- ✅ 已验收
- ⚠️ 验收不通过

## 七、典型场景示例

### 场景：客户询盘 → 方案报价

**流程：**

```
1. 运营主管接收客户询盘
   ↓
2. 判断需要营收主管参与（定价）
   ↓
3. 发起跨部门交接：
   - 客户信息包
   - 产品需求
   - 期望报价时间
   ↓
4. 营收主管接收
   ↓
5. 营收主管完成成本核算+报价方案
   ↓
6. 反馈运营主管
   ↓
7. 运营主管协调营销主管完成方案美化
   ↓
8. 运营主管交付客户
   ↓
9. 客户反馈
   ↓
10. 闭环记录
```

---

_本约定为 v1.0，所有跨部门任务必须遵循此流程。_
