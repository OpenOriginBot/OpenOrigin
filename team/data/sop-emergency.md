# 跨部门协作SOP

## 一、断货应急流程

### 触发条件
库存=0且存在未发货订单

### 处理流程
```
1. 运营部（ops-head）检测到断货风险
   → 立即通知营收部（rev-head）提供备货方案
   → 同步通知营销部（mkt-head）暂停推广

2. 营收部（rev-head）
   → 30分钟内提供备用供应商库存
   → 给出成本差异评估

3. 营销部（mkt-head）
   → 调整客服自动回复话术
   → 准备客户安抚方案

4. CEO统一决策
   → 确认执行方案
   → 对外通知客户
```

### 时间要求
- 检测到断货：立即
- 营收部响应：30分钟内
- 营销部响应：1小时内
- 整体决策：2小时内

## 二、差评危机流程

### 触发条件
单日差评≥3条，或评分可能降至4.7以下

### 处理流程
```
1. 营销部（mkt-head）
   → 立即识别差评内容
   → 分析差评原因（产品/物流/服务）
   → 提供3个回复方案

2. CEO确认
   → 选择回复策略
   → 授权补偿方案（如需）

3. 运营部（ops-head）协作
   → 如因物流问题，配合核查物流信息
   → 如因产品问题，通知营收部评估

4. 营收部（rev-head）协作
   → 评估是否需要更换供应商
   → 成本核算（如需补偿）
```

### 时间要求
- 差评识别：1小时内
- 回复方案：2小时内
- 公开回复：24小时内

## 三、大促活动流程

### 触发条件
平台大促（618/双11等）或自主促销活动

### 处理流程
```
1. 营收部（rev-head）主导
   → 成本核算与定价策略
   → 优惠券额度申请
   → 备货量评估

2. 运营部（ops-head）协作
   → 库存保障计划
   → 发货能力评估
   → 物流跟踪加强

3. 营销部（mkt-head）主导
   → 活动页面设计
   → 推广计划制定
   → 客服话术准备

4. CEO统一审批
   → 最终定价确认
   → 活动规则审批
```

### 时间要求
- 大促前7天：完成各部门方案
- 大促前3天：CEO审批
- 大促前1天：最终检查

## 四、新品上架流程

### 触发条件
引入新SKU或新供应商产品

### 处理流程
```
1. 营收部（rev-head）评估
   → 供应商资质审核
   → 成本与利润测算
   → 最低定价建议

2. 运营部（ops-head）评估
   → 库存备货计划
   → 物流仓储安排

3. 营销部（mkt-head）准备
   → 产品详情页文案
   → 主图与视频素材
   → SEO关键词优化

4. CEO审批
   → 确认上新
   → 授权售价范围
```

### 时间要求
- 供应商评估：3天内
- 内容准备：5天内
- CEO审批：1天内

---

## 五、周度同步会议（虚拟）

### 时间
每周一 09:00（可根据情况调整）

### 参与方
CEO主持，三位部门主管汇报

### 汇报模板

**运营部：**
- 上周发货及时率
- 库存健康状态
- 本周重点项目

**营销部：**
- 上周转化率/访客数
- 差评处理情况
- 本周推广计划

**营收部：**
- 上周毛利率/GMV
- 供应商绩效
- 成本优化建议

_本SOP为跨部门协作基准，实际执行可根据具体情况进行微调。_
