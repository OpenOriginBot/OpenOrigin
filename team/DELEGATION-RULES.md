# DELEGATION-RULES.md - 流转分配清单

> 定义工作在多智能体团队中的流转规则，确保任务精准分发、正常处理、结果沿委派链路回流闭环。

---

## 一、任务分类与主管归属

| 任务类型 | 负责主管 | 优先级 | 处理时效 |
|----------|----------|--------|----------|
| 客户交付成果（订单/发货/售后） | ⏱️ 运营主管 | 🔴 高 | 即时响应，4小时内处理 |
| 内容创作（文案/主图/详情页） | ✨ 营销主管 | 🔴 高 | 24小时内初稿 |
| 客源开发触达（外联/开发信） | ✨ 营销主管 | 🟡 中 | 48小时内执行 |
| 技术问题修复（系统/接口故障） | ⏱️ 运营主管 | 🔴 高 | 即时响应，2小时内定位 |
| 方案初稿撰写（报价/提案） | 💰 营收主管 | 🟡 中 | 48小时内初稿 |
| 数据汇报（日报/周报/月报） | ⏱️ 运营主管 | 🟡 中 | 定时自动生成 |
| 定价策略调整 | 💰 营收主管 | 🔴 高 | 24小时内分析建议 |
| 供应商谈判跟进 | 💰 营收主管 | 🟡 中 | 72小时内完成 |
| 竞品分析调研 | ✨ 营销主管 | 🟢 低 | 5个工作日内 |
| 毛利率异常排查 | 💰 营收主管 | 🔴 高 | 4小时内排查 |

## 二、委派时必须附带的信息

### 标准委派信息包

```
任务编号：[自动生成]
任务类型：[从上述分类选择]
任务描述：[清晰明确的工作内容]
负责主管：[归属的主管ID]
开始时间：[YYYY-MM-DD HH:mm]
截止时间：[YYYY-MM-DD HH:mm]
期望交付物：[具体产出物描述]
验收标准：[判定办结的依据]
上级确认：[是否需要人工审批]
关联任务：[如有前置/关联任务]
背景资料：[必要上下文信息]
```

### 分级信息包

**完整包（红色权限任务）：**
- 必须包含：全部字段
- 必须附上：预估成本/风险评估

**标准包（黄色权限任务）：**
- 必须包含：任务描述、截止时间、验收标准
- 可省略：成本/风险评估

**精简包（绿色权限任务）：**
- 可仅包含：任务描述、截止时间

## 三、各类任务详细处理规则

### 🔴 客户交付成果 → ⏱️ 运营主管

**定义：** 涉及客户订单处理、发货跟进、售后响应的任务。

**委派必须附带：**
- 订单编号
- 客户要求描述
- 紧急程度评级

**处理时效：**
| 类型 | 时效要求 |
|------|----------|
| 催发货 | 1小时内响应 |
| 退款申请 | 4小时内审核 |
| 差评公关 | 2小时内处理 |
| 物流异常 | 4小时内跟进 |

**遇阻上报：**
1. 客户情绪激烈升级
2. 退款金额超权限
3. 供应商无法配合

**审批要求：**
- 单笔退款 ≥500元需人工审批
- 差评公关方案需事后确认

---

### ✨ 内容创作 → ✨ 营销主管

**定义：** 文案撰写、主图设计、详情页优化、种草内容创作。

**委派必须附带：**
- 产品信息
- 目标受众
- 渠道要求
- 参考案例（如有）

**处理时效：**
| 类型 | 时效要求 |
|------|----------|
| 主图优化 | 48小时内3个方案 |
| 详情页文案 | 72小时内初稿 |
| 推广文案 | 24小时内 |

**遇阻上报：**
1. 产品信息不完整
2. 多部门协调需求
3. 超出预算

**审批要求：**
- 大促文案需事前确认
- 新品上架方案需事后汇报

---

### ✨ 客源开发触达 → ✨ 营销主管

**定义：** 外联邮件发送、客户开发信、cold outreach。

**委派必须附带：**
- 目标客户名单
- 产品/服务信息
- 发送数量上限
- 个性化要求

**处理时效：**
- 单批次 ≤50封
- 间隔 ≥2小时发送

**遇阻上报：**
1. 邮件发送被拦截
2. 客户大量投诉

**审批要求：**
- 新增客户名单需确认
- 超过500封/批次需事前审批

---

### ⏱️ 技术问题修复 → ⏱️ 运营主管

**定义：** 系统故障、接口异常、数据错误。

**委派必须附带：**
- 问题描述
- 影响范围
- 紧急程度
- 发现时间

**处理时效：**
| 级别 | 时效要求 |
|------|----------|
| P0（系统崩溃） | 30分钟内响应 |
| P1（功能异常） | 2小时内定位 |
| P2（轻微故障） | 8小时内处理 |

**遇阻上报：**
1. 问题根源不明
2. 需要供应商介入
3. 需要数据恢复

**审批要求：**
- 数据变更需事后确认
- 系统重启需记录

---

### 💰 方案初稿撰写 → 💰 营收主管

**定义：** 报价方案、项目提案、合作协议初稿。

**委派必须附带：**
- 客户基本信息
- 需求概要
- 预算范围（如有）
- 截止时间

**处理时效：**
- 报价方案：48小时内
- 项目提案：72小时内
- 紧急方案：24小时内

**遇阻上报：**
1. 客户需求超出服务范围
2. 预算明显低于成本
3. 需要法务审核

**审批要求：**
- 报价方案需事前确认
- 合作协议需法务审核

---

### ⏱️ 数据汇报 → ⏱️ 运营主管

**定义：** 日报、周报、月报的自动生成。

**触发方式：**
- 定时任务自动生成
- 或人工指令触发

**处理时效：**
| 类型 | 生成时间 |
|------|----------|
| 日报 | 每日 09:00 |
| 周报 | 每周一 10:00 |
| 月报 | 每月1日 09:00 |

**交付物：**
- Markdown 格式报告
- 数据可视化摘要
- 异常预警清单

**遇阻上报：**
- 数据源异常
- 报表生成失败

**审批要求：**
- 自动执行，事后查阅

## 四、遇阻上报流程

```
┌─────────────────────────────────────────────────┐
│                     任务遇阻                      │
└─────────────────────┬───────────────────────────┘
                      │
          ┌──────────┴──────────┐
          │                     │
      阻塞可解决              阻塞无法解决
          │                     │
    ┌─────┴─────┐         ┌────┴────┐
    │           │         │         │
  等待资源   协调内部   升级上报   记录挂起
    │           │         │         │
    └─────┬─────┘     ┌────┴────┐  │
          │             │         │  │
       重试执行      主控介入   人工处理  标注延期
                          │
                    ┌────┴────┐
                    │         │
                决策变更    外部介入
```

**上报模板：**
```
【遇阻上报】
任务：[任务编号+名称]
负责主管：[主管名称]
阻塞原因：[详细描述]
已尝试方案：[已尝试的解决方式]
影响评估：[对业务的影响]
建议方案：[2-3个可选建议]
需协助事项：[具体需要什么帮助]
```
