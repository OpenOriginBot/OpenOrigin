| **负责人** | [待定] |

---

## 用途

根据客户需求自动生成完整的解决方案文档和报价方案。适用于客户询盘回复、项目提案、招标应答、合作伙伴提案等场景。输出内容包含需求分析、解决方案、产品/服务说明、报价明细、合作条款等模块。

## 输入

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `customer_name` | string | 是 | - | 客户名称 |
| `customer_need` | string | 是 | - | 客户需求描述 |
| `product_type` | string | 是 | - | 产品/服务类型 |
| `budget_range` | string | 否 | - | 预算范围 |
| `proposal_type` | string | 否 | solution-proposal | 方案类型 |
| `timeline` | string | 否 | - | 期望交付时间 |
| `include_pricing` | boolean | 否 | true | 是否包含报价 |

**示例输入：**
```json
{
  "customer_name": "广州某科技公司",
  "customer_need": "需要一套智能灯带解决方案，用于产品展示厅氛围营造",
  "product_type": "智能灯带+控制系统",
  "budget_range": "5-10万",
  "proposal_type": "solution-proposal",
  "timeline": "2周内交付",
  "include_pricing": true
}
```

## 约束规则

### 运行时限制
- 最大执行时间：`60 秒`
- 最大内存：`1024 MB`
- 最大调用频率：`30 次/小时`

### 内容限制
- ❌ 禁止虚构不存在的功能/服务
- ❌ 禁止出具无依据的价格承诺
- ❌ 禁止包含排他性条款（除非明确授权）
- ✅ 报价必须标注有效期
- ✅ 必须包含免责条款

### 格式要求
- 方案标题需包含客户名称
- 必须包含：执行摘要、需求分析、解决方案
- 报价部分需分项列示

---

## 故障模式

| 故障 | 原因 | 处理方式 |
|------|------|----------|
| `E_MISSING_REQUIRED` | 缺少必填参数 | 返回缺失字段列表 |
| `E_SCOPE_CREEP` | 需求描述过长过泛 | 建议拆分为多个方案 |
| `E_PRICE_UNAVAILABLE` | 无法估算价格 | 生成无报价版本，标注需人工确认 |
| `E_TIMEOUT` | 方案过长超时 | 生成精简版，标注完整版需单独请求 |

## 方案类型说明

| 类型 | 用途 | 特点 |
|------|------|------|
| `solution-proposal` | 综合解决方案 | 含需求分析+方案+报价 |
| `quotation` | 纯报价单 | 简化的产品+价格列表 |
| `project-proposal` | 项目提案 | 含时间线+里程碑+风险评估 |
| `partnership-proposal` | 合作提案 | 含双方权责+合作模式 |
