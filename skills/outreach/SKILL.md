## 用途

自动生成客户外联邮件和消息，支持个性化开场白、价值主张和行动号召。可应用于新客户拓展、老客户激活、合作伙伴联系等场景。

---

### 自动触发
- [ ] 对话中检测到关键词：`外联`、`联系客户`、`开发信`、`cold-email`

### 手动触发
- [x] 用户指令：`写一封外联邮件` / `生成客户开发信`


## 输入

| 参数 | 类型 | 必填 | 默认值 | 说明 |
|------|------|------|--------|------|
| `recipient_name` | string | 是 | - | 收件人姓名 |
| `recipient_company` | string | 否 | - | 收件人公司 |
| `product` | string | 是 | - | 推广产品/服务 |
| `goal` | string | 是 | - | 目标：询盘/成交/合作/活动 |
| `tone` | string | 否 | professional | 语气：professional/friendly/formal |
| `language` | string | 否 | zh-CN | 语言 |

**示例输入：**
```json
{
  "recipient_name": "李总",
  "recipient_company": "深圳市某电商公司",
  "product": "智能灯带",
  "goal": "询盘转化",
  "tone": "professional",
  "language": "zh-CN"
}
```

## 约束规则

### 运行时限制
- 最大执行时间：`30 秒`
- 最大内存：`512 MB`
- 最大调用频率：`50 次/小时`

### 内容限制
- ❌ 禁止生成虚假夸大宣传
- ❌ 禁止使用绝对化用语（"最好"、"第一"等）
- ❌ 禁止涉及政治敏感内容
- ❌ 禁止泄露客户隐私信息
- ✅ 所有输出建议标注"仅供参考"

### 质量要求
- 邮件长度控制在 150-500 字
- 必须包含称呼、正文、落款、行动号召

---

## 故障模式

| 故障 | 原因 | 处理方式 |
|------|------|----------|
| `E_TIMEOUT` | 输入过大或模型响应慢 | 减少产品描述长度，重试 |
| `E_EMPTY_OUTPUT` | 无法生成有效内容 | 返回错误，建议简化输入 |
| `E_TOO_LONG` | 生成内容超出限制 | 截断并返回前500字 |
